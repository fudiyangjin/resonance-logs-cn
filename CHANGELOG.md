# Changelog v0.0.5

## Logs

**buff 相关**

- 独立模式不再显示未选中的 buff, 特殊 buff 不选择就不显示

**dps 相关**

- 重置逻辑修改
  - 移除根据副本状态判断重置的逻辑, 避免 bkl 随意下发副本状态(比如协会狩猎单 boos 死亡就下发副本结束)
  - 设置重置标记异步延迟 3s, 避免多余伤害包下发生成多段垃圾历史记录
  - 增加对应配置, 配置需要跳过的任务 id, 避免多 P 战斗有多个目标 id 导致中途重置(比如机器人团本会多次下发消灭 boss 目标, 2P 会切换成解码目标)
  - 对于现在的重置逻辑应该是: 大师本自动切分道中和 boss, 历史记录有 2 条; 团本每个 boss 独立切分; 团灭自动重置; 对于同一场景的重置均是下次攻击目标后自动重置(大师本清完道中, 攻击 boss 后自动重置 | 团本每次攻击新 boss 时自动重置 | 团灭后下次攻击 boss 时自动重置)
- 重构查询场景的逻辑, 大师本拼接难度
- 历史记录每场战斗的列表页显示玩家 + 职业图标
- 支持显示赛季强度(DPS检测-设置-通用设置)

**模组相关**

- 自身信息立即刷入数据库, 避免模组获取不到最新

**角色面板属性相关**

- 技能监控页新增角色面板属性监控(技能监控-角色面板属性监控)

**注意**

- 如果之前使用过 0.0.2/0.0.3 第一次使用 0.0.5-beta 需要清理 `%LOCALAPPDATA%\resonance-logs-cn` 目录下的 `resonance-logs-cn.db` 再重新启动应用
