# Resonance Logs - Global

A desktop combat data analysis app for [Blue Protocol: Star Resonance](https://www.starresonance.com/), featuring real-time DPS tracking, Buff monitoring, skill cooldown display, and more.

This project is modified from [resonance-logs](https://github.com/resonance-logs/resonance-logs)

## Features

- **DPS Tracking**: Real-time DPS statistics, overall active time, and history logs
- **Buff Monitoring**: Skill cooldowns, Buff tracking, Buff aliases, and quick category monitoring (Food/Alchemy)
- **Module Optimization**: Uses packet-captured module data to intelligently calculate optimal module combinations (with GPU acceleration support)
- **Custom Panels**: Displays counters and Buff monitors as progress bars
- **In-Game Overlay**: Transparent always-on-top window with masking support and hotkey toggling
- **Auto Update**: Supports in-app OTA updates

## Tech Stack

- **Backend**: Rust + [Tauroh godi 2](https://tauri.app/)
- **Frontend**: SvelteKit 5 + Svelte + TypeScript + Tailwind CSS
- **Data Capture**: WinDivert / Npcap (Windows packet capture)

## System Requirements

- **Platform**: Windows (administrator privileges required to use WinDivert)
- **Node.js**: Required for frontend builds
- **Rust**: Required for Tauri builds

## Quick Start

### Install Dependencies

```bash
npm install
```

### Development Mode

```bash
npm run tauri dev
```

### Build

```bash
npm run tauri build
```

Build output is generated by default in `src-tauri/target/release/bundle/`, including NSIS installer packages.

### Module Optimization Build Notes

The module optimization feature depends on a C++ extension and can optionally use GPU acceleration through CUDA/OpenCL. Before building, please refer to the environment requirements of [StarResonanceAutoMod](https://github.com/fudiyangjin/StarResonanceAutoMod):

**Basic Requirements (CPU version):**

- **Visual Studio Build Tools 2019/2022** or Visual Studio (MSVC compiler)
- Windows SDK

**Additional Requirements for GPU version:**

- **CUDA Toolkit 12.9** (for NVIDIA GPUs, RTX 20XX and above; version 12.9+ supports builds for the 50XX series)
- or **OpenCL** (for NVIDIA/AMD/Intel GPUs, usually provided by graphics drivers or CUDA)

The build script will automatically detect CUDA/OpenCL. If neither is detected, only the CPU version will be compiled. If the C++ source directory `src-tauri/src/module_optimizer/cpp/` does not exist, the module optimization module build will be skipped, while all other features remain unaffected.
