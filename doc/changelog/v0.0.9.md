# Changelog v0.0.9

## 更新内容

### DPS 相关

- 实时, 历史增加显示数据小数位的设置(DPS检测-设置-实时/历史-通用设置)
- 增加配置项切换到 w/亿(DPS检测-设置-实时/历史-通用设置)
- 修复极简主题下打桩模式按钮不自动隐藏的问题
- 移动是否显示打桩模式按钮设置到标题栏设置(DPS检测-主题-实时-标题栏设置), 选择打桩目标还在原位
- 怪物死亡时不再推送对应 boss 血量
- 实时 dps 中的治疗技能数据按路由进行聚合
- 修改重置逻辑, 切图后实时 dps 不清空, 下次有伤害包数据后自动清理(界面), 历史记录还是会正常在切图时入库. 再次重申下现在的重置逻辑:
  - 大师本自动切分道中和 boss, 历史记录有 2 条;
  - 团本每个 boss 独立切分;
  - 团灭自动重置;
  - 对于同一场景的重置均是下次攻击目标后自动重置(大师本清完道中, 攻击 boss 后自动重置 | 团本每次攻击新 boss 时自动重置 | 团灭后下次攻击 boss 时自动重置)
  - 本次新增:切图后实时 dps 不清空, 下次有伤害包数据后自动清理(界面)
- 打桩模式 UI 响应优化
- live 界面技能详情点击响应优化
- 新增有效治疗和有效 hps 的展示

### 实时监控相关

- 怪物死亡时不再推送怪物的仇恨, buff 等信息
- 无穷雷充能层数调整为 2, 增加居合天赋大招 cd

### 模组相关

- 支持 5 模组非 NVIDIA 显卡的 GPU 运算

## 注意事项

### 兼容性提醒

- 如果之前使用过 0.0.2 或 0.0.3，第一次使用 0.0.4-0.0.9 需要清理 `%LOCALAPPDATA%\resonance-logs-cn` 目录下的 `resonance-logs-cn.db`，再重新启动应用
- 由于已经优化了 npcap 的处理逻辑，请优先使用 npcap 进行抓包，避免因 WinDivert 驱动签名过期引发杀毒软件误报、误隔离，导致应用无法正常运行
- 首次使用 0.0.8 时需要双击实时监控的"启用实时监控"和怪物监控的"启用怪物监控", 即取消后重新启用, 用于持久化聚合后的监听列表, 解决应用启动时监听列表发送不稳定的问题
- 0.0.9 由于新增了有效治疗字段, 旧的历史记录报错属于正常现象, 新的历史记录正常
