# Changelog v0.1.1

## 更新内容

### DPS 相关

- live 界面支持紧凑模式展示实时数据(隐藏表头仅展示占比, 秒伤, 总伤) (主题-通用-紧凑模式)
  - 名称-职业      总伤(秒伤) 占比
- 支持 2s 数据的死亡回放, 用于了解死亡前都受到了哪些伤害
  - 对于 live 界面, 需要手动开启(主题-实时-导航标签)
  - 对于历史记录在 tank 标签后面
- 支持 live 界面的 tank 标签点击具体角色后和历史记录一样展示对应详情
- 支持 live, 历史记录界面展示受到伤害的元素类型, 物理还是魔法
- 新增了一些 damageid

### 实时监控相关

- 新增格挡天赋-岩之心触发计数(实时监控-自定义监控-添加计数器)
- 新增赤红天赋-迷狂劈杀连招计数(实时监控-自定义监控-添加计数器)
- 修复冰法 X3 真实因子在大招区域内可能不会重置的问题
- 全量同步时增加解析相关属性
  - 修复资源监控必须切职业才生效的问题
  - 避免依靠属性变更的状态机发生异常

### 怪物监控相关

- 怪物监控 buff 支持优先级设置

## 注意事项

### 兼容性提醒

- 如果之前使用过 0.0.2 或 0.0.3，第一次使用 0.0.4-0.1.1 需要清理 `%LOCALAPPDATA%\resonance-logs-cn` 目录下的 `resonance-logs-cn.db`，再重新启动应用
- 由于已经优化了 npcap 的处理逻辑，请优先使用 npcap 进行抓包，避免因 WinDivert 驱动签名过期引发杀毒软件误报、误隔离，导致应用无法正常运行
- 首次使用 0.0.8 时需要双击实时监控的"启用实时监控"和怪物监控的"启用怪物监控", 即取消后重新启用, 用于持久化聚合后的监听列表, 解决应用启动时监听列表发送不稳定的问题

### 特别注意

- 如果你自己修改了代码又不想提交 PR，在分享给其他人的时候请把应用名、版本号等原版相关信息都改掉，不要带着原仓库的信息，尤其是应用名和版本号
- 常见问题见使用手册中的 hmtl
