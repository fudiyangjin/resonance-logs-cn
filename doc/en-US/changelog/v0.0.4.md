# Changelog v0.0.4

## Logs

**Buff-related**

- Buff grouped display, monitor all buffs, buff priority ordering
- Removed talent-linked buff logic; manual buff_name concatenation (reconfigure if monitoring talent buffs)
- Distinguish user selection vs default listen items; only show user-selected items
- Reset no longer clears buff settings—avoids buff display interruption

**DPS-related**

- Refactored multiple areas, simplified processing
- Refactored SQLite operations; several tables removed—delete `resonance-logs-cn.db` under `%LOCALAPPDATA%\resonance-logs-cn` and restart
- Added instance packet parsing; resets based on target transitions and wipe buffs. Same scene: delayed reset (clear trash → reach Boss → auto reset on first Boss hit; master wipe → reset on next Boss hit)
- History shows DPS breakdown per target. Healing per character shows skill distribution plus healing to teammates
- Further display of each skill’s damage ID composition
- Fixed frontend hide name, show class/spec, etc.
- Fixed font settings

## Notes

- If you used 0.0.2/0.0.3, first launch on 0.0.4 requires deleting `resonance-logs-cn.db` under `%LOCALAPPDATA%\resonance-logs-cn` and restarting
- Reconfigure if monitoring talent buffs
