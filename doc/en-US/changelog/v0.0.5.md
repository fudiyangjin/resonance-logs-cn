# Changelog v0.0.5

## Logs

**Buff-related**

- Standalone mode no longer shows unselected buffs; special buffs hidden unless selected

**DPS-related**

- Reset logic changes
  - Removed reset based on instance state—avoids bad instance state from server (e.g. guild hunt single boss death sends instance end)
  - Reset flag set asynchronously with 3s delay—avoids junk history from extra damage packets
  - Config to skip quest IDs—avoids mid-fight reset with multiple target IDs in multi-phase fights (e.g. robot raid boss kill objectives, 2P decode target switch)
  - Current reset logic: master dungeon auto-splits trash and Boss (2 history entries); raid splits per Boss; wipe auto-reset; same-scene resets on next attack (trash cleared → first Boss hit | new Boss in raid | post-wipe Boss hit)
- Refactored scene query logic; master dungeon appends difficulty
- History list page shows player + class icon per fight
- Season strength display (DPS Meter → Settings → General Settings)

**Module-related**

- Flush self info to DB immediately—modules get latest data

**Character panel-related**

- Character panel attribute monitoring on skill monitor page (Live Monitor → Character Panel)

**Notes**

- If you used 0.0.2/0.0.3, first launch on 0.0.5-beta requires deleting `resonance-logs-cn.db` under `%LOCALAPPDATA%\resonance-logs-cn` and restarting
