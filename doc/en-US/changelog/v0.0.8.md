# Changelog v0.0.8

## Changes

### DPS

- Added s3 data on top of YozoraHoshi data
- Dual-axe class icon and specialization name display
- Fixed wrong share for boss multi-target skills
- Fixed history search bar reset when switching filter columns
- History search criteria persistence
- Fixed history scene query; unified label “Scene”
- History overview shows favorites
- Rebuild history excludes favorited records
- Fixed abnormal reset in Mechanized Foundry boss phase
- New dummy mode
  - Select dummy type on live page (DPS Meter → Live → Dummy mode)
  - Start enters dummy standby
  - Your first damage starts the dummy session
  - Records only that dummy’s damage; teammate damage to same dummy is OK
  - Auto reset when time ends; view breakdown in history
  - Manual reset returns to standby for next trigger
  - Leaving guild, zone change, etc. auto-cancels dummy mode

### Live Monitor

- More stable initialization for listen lists
- Custom monitor supports groups with separate display
- Added Thunder Shadow Swordsman skills and resources
- Skill duration settings and display; default includes Scythe Cart
- Optimized live monitor rendering—no full overlay recalc every frame

### Monster Monitor

- Configurable threat list display count
- Added `ExtraBuffMonitoredMonsters.json` for non-boss buff monitoring; default 2 elite dummies
- New dummy mode

### Module Calculator

- Refactored enumeration algorithms
  - 4-module CPU enum: 87s → 43s for 800 modules
  - CUDA 1000-module enum: 23s → 3s
- 5-module filtering support
- Improved module result display
- Fixed CUDA context not released (VRAM not freed)
- Module sum filter; filter modules with only 1 attribute

## Notes

### Compatibility

- If you used 0.0.2 or 0.0.3, first launch on 0.0.8 requires deleting `resonance-logs-cn.db` under `%LOCALAPPDATA%\resonance-logs-cn` and restarting
- Prefer Npcap; WinDivert signature expiry may cause AV false positives
- Non-NVIDIA GPUs cannot use enumeration strategy for module quality supplement yet
- First launch on 0.0.8: toggle **Enable Live Monitor** and **Enable Monster Monitor** off then on to persist aggregated listen lists and fix unstable listen list on startup

- If you fork without submitting a PR, change app name, version, and upstream branding before sharing
