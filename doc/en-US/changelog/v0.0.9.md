# Changelog v0.0.9

## What's New

### DPS-related

- Live and history: setting for decimal places in displayed data (DPS Meter → Settings → Live / History → General Settings)
- Option to switch abbreviation to Chinese (12.3w / 1.2亿) (DPS Meter → Settings → Live / History → General Settings)
- Fixed training dummy button not auto-hiding under minimal theme
- Moved “show training dummy button” to **Header Settings** (DPS Meter → Themes → Live → Header Settings); dummy target selection unchanged
- No longer push Boss HP when monster dies
- Live DPS healing skill data aggregated by route
- Reset logic change: Live DPS no longer clears on zone change; UI clears on next damage packet; history still saves on zone change. Current reset logic recap:
  - Master dungeon auto-splits trash and Boss (2 history entries)
  - Raid splits per Boss
  - Wipe auto-reset
  - Same-scene resets on next attack (trash cleared → first Boss hit | new Boss in raid | post-wipe Boss hit)
  - **New in this release**: Live DPS not cleared on zone change; UI clears on next damage packet
- Training dummy mode UI responsiveness improved
- Live skill detail click responsiveness improved
- Added effective healing and effective HPS display

### Live Monitor-related

- No longer push monster threat, buff, etc. when monster dies
- Infinite Thunder charge stacks set to 2; added Iai talent ultimate CD

### Module-related

- GPU compute for 5-module setups on non-NVIDIA GPUs

## Notes

### Compatibility

- If you used 0.0.2 or 0.0.3, first use of 0.0.4–0.0.9 requires deleting `resonance-logs-cn.db` under `%LOCALAPPDATA%\resonance-logs-cn` and restarting
- Npcap handling improved—prefer Npcap; WinDivert driver signature expiry may cause AV false positives and block the app
- First use of 0.0.8: toggle **Enable Live Monitor** on Live Monitor and **Enable Monster Monitor** on Monster Monitor off and on to persist aggregated watch lists and fix unstable watch list on startup
- 0.0.9 adds effective healing fields—errors on old history entries are expected; new history works normally
