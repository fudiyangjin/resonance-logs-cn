# Changelog v0.1.0

## What's New

### DPS-related

- Updated s3 config, partial scene renames @YozoraHoshi

### Live Monitor-related

- Custom monitor: factor energy tracking (Live Monitor → Custom Monitor → Custom Counter Rules → New Rule → select multiple Sources and Slots → name and save → Add Counter and pick corresponding Slot)
  - Added Iai inspiration factors X1, X2, X4, X5; true factors X1, X2
  - Added dual-axe inspiration factors X1, X2, X4, X9, X8; true factors X1, X3, X4, X5
  - Added guitar inspiration factors X10, X9, X5, X11; true factor X5
  - Added hawk bow inspiration factors X7, X9, X11; true factors X5, X6
  - Added ice mage inspiration factors X1, X3, X4, X6, X7, X10; true factors X1, X3, X4, X5
  - See Live Monitor in the user manual for setup details
- New preset monitor: Flame Horn – Passive
  - Slot 1: damage specialization counter—counts remaining Absolute Luck trigger opportunities
  - Slot 2: Absolute Luck CD—~15 s steady, ~5 s when active skill is used
  - Recommended: also track “Flame Horn – Passive – Luck” (Absolute Luck buff), Slot 1, Slot 2 in the same area; see Live Monitor in the user manual
- Buffs with 1 stack and 0 duration display normally when user adds them to watch list
- Character panel: elemental damage attribute monitoring
- Fixed character resource display after update—push by resource index ID

## Notes

### Compatibility

- If you used 0.0.2 or 0.0.3, first use of 0.0.4–0.1.0 requires deleting `resonance-logs-cn.db` under `%LOCALAPPDATA%\resonance-logs-cn` and restarting
- Npcap handling improved—prefer Npcap; WinDivert driver signature expiry may cause AV false positives and block the app
- First use of 0.0.8: toggle **Enable Live Monitor** on Live Monitor and **Enable Monster Monitor** on Monster Monitor off and on to persist aggregated watch lists and fix unstable watch list on startup
