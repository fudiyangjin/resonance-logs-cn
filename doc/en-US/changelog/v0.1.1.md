# Changelog v0.1.1

## What's New

### DPS-related

- Live UI supports **Compact Mode** for real-time data (hide header, show share %, DPS, total damage) (DPS Meter → Themes → General → Compact Mode)
  - Name–Class      Total (DPS) Share %
- 2-second death replay to see damage taken before death
  - Live: enable manually (DPS Meter → Themes → Live → navigation tabs)
  - History: after Tank tab
- Live Tank tab: click a character for details like history
- Live and history show damage element type (physical vs magic)
- Added some damage IDs

### Live Monitor-related

- New Block talent – Rock Heart trigger counter (Live Monitor → Custom Monitor → Add Counter)
- New Crimson talent – Frenzy Cleave combo counter (Live Monitor → Custom Monitor → Add Counter)
- Fixed ice mage X3 true factor possibly not resetting inside ultimate area
- Full sync now parses related attributes
  - Fixed resource monitor requiring class switch to take effect
  - Avoids state machine errors relying on attribute changes

### Monster Monitor-related

- Monster Monitor buff priority settings

## Notes

### Compatibility

- If you used 0.0.2 or 0.0.3, first use of 0.0.4–0.1.1 requires deleting `resonance-logs-cn.db` under `%LOCALAPPDATA%\resonance-logs-cn` and restarting
- Npcap handling improved—prefer Npcap; WinDivert driver signature expiry may cause AV false positives and block the app
- First use of 0.0.8: toggle **Enable Live Monitor** on Live Monitor and **Enable Monster Monitor** on Monster Monitor off and on to persist aggregated watch lists and fix unstable watch list on startup
