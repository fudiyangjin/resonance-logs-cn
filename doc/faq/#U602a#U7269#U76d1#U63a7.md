# FAQ · 怪物监控

## `ExtraBuffMonitoredMonsters.json` 是做什么的？

应用在处理附近实体的 **Buff 同步数据** 时，默认只会对 **Boss** 目标解析并更新「怪物 Buff」类监控
`src-tauri/meter-data/ExtraBuffMonitoredMonsters.json` 用来 **按怪物模板 ID 扩展** 这一范围：列表里的 ID 即使不是 Boss，也会同样参与上述 Buff 解析，便于在木桩、精英等非 Boss 目标上调试或查看 Buff。

### 文件格式

与仓库中示例一致，顶层字段为 `monsterIds`，值为正整数数组（对应游戏内的怪物模板 ID，与 `MonsterIdNameType.json` 中的键一致）：

```json
{
  "monsterIds": [115, 122]
}
```

### 默认内容含义

当前默认包含 **115**、**122**，在 `MonsterIdNameType.json` 中分别对应 **精英敌方木桩**、**精英守护木桩**(均为非 Boss 的精英木桩类目标).

修改后需 **重启应用**；若自行增删 ID，请确认该 ID 在 `MonsterIdNameType.json` 中存在且确实是你需要监听 Buff 的怪物模板。请及时备份, 以免版本更新后被覆盖
