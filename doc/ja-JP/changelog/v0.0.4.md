# Changelog v0.0.4

## Logs

**buff 関連**

- Buff グループ表示、すべての Buff 監視、Buff 優先度ソートをサポート
- 天赋関連 Buff のロジックを削除。buff_name を手動連結（天赋類 Buff を監視する場合は再設定が必要）
- ユーザー選択とデフォルト監視項目を区別し、余分な表示を避け、ユーザー選択のみ表示
- リセット時に Buff 関連設定をクリアしない（Buff 表示の中断・再表示不能を防止）

**dps 関連**

- 複数箇所をリファクタし処理を簡素化
- sqlite 全体操作をリファクタ。複数テーブル削除のため、`%LOCALAPPDATA%\resonance-logs-cn` 内の `resonance-logs-cn.db` を削除してから再起動が必要
- ダンジョン情報パケット解析を追加。ターゲット変換とワイプ Buff に基づく主リセット。同一シーンでは遅延リセット（道中クリア→Boss 到達後 Boss 攻撃で自動リセット；マスター本ワイプ→次回 Boss 攻撃で自動リセット）
- 履歴にターゲット別 DPS 分布を追加。回復は単キャラで回復スキル分布に加え味方への回復量分布も表示
- 各スキルのダメージ ID 構成をさらに表示
- フロントの名称非表示、職業特化表示などを修正
- フォント設定を修正

## 注意

- 以前 0.0.2/0.0.3 を使用していた場合、初めて 0.0.4 を使うときは `%LOCALAPPDATA%\resonance-logs-cn` 内の `resonance-logs-cn.db` を削除してから再起動
- 天赋類 Buff を監視している場合は再設定が必要
