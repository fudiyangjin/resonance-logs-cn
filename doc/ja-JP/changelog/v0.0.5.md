# Changelog v0.0.5

## 変更ログ

**Buff 関連**

- 独立モードで未選択の Buff を表示しないよう変更。特殊 Buff も未選択なら非表示

**DPS 関連**

- リセットロジックを変更
  - ダンジョン状態によるリセット判定を削除。bkl が任意にダンジョン終了状態を送信する問題を回避（例：ギルド狩猟で単一 Boss 死亡時にダンジョン終了を送信）
  - リセットフラグを非同期で 3 秒遅延。余分なダメージパケットによるゴミ履歴の複数生成を防止
  - スキップするクエスト ID を設定可能に。マルチ P 戦闘で複数ターゲット ID による途中リセットを回避（例：ロボットレイドで Boss 撃破目標が複数回送信、2P でデコード目標へ切り替わる）
  - 現在のリセットロジック：マスターダンジョンは道中と Boss を自動分割、履歴 2 件。レイドは Boss ごとに独立分割。ワイプで自動リセット。同一シーン内のリセットはすべて次の攻撃ターゲット後に自動（マスターダンジョン：道中クリア後 Boss 攻撃でリセット | レイド：新 Boss 攻撃ごと | ワイプ後：次の Boss 攻撃でリセット）
- シーン照会ロジックを再構成。マスターダンジョンに難易度を連結
- 履歴の各戦闘リストページにプレイヤー + 職業アイコンを表示
- シーズン強度の表示をサポート（DPSメーター-設定-共通設定）

**モジュール関連**

- 自身情報を即座に DB へ書き込み。モジュールが最新情報を取得できない問題を回避

**キャラクターパネル属性関連**

- スキル監視ページにキャラクターパネル属性監視を追加（スキル監視-キャラクターパネル属性監視）

**注意**

- 0.0.2/0.0.3 を使用していた場合、0.0.5-beta を初めて使うときは `%LOCALAPPDATA%\resonance-logs-cn` 内の `resonance-logs-cn.db` を削除してからアプリを再起動してください
