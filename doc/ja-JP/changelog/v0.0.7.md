# Changelog v0.0.7

## 変更ログ

**DPS 関連**

- npcap が一部シーンで正常にキャプチャできない問題を修正。イーサネットフレーム処理範囲を拡張
- s3 データを追加。一部データをローカライズ @YozoraHoshi
- 意味のない Boss 死亡メッセージの送信を削除
- 全量同期前に一部データをクリーンアップ。リストを整理
- `MonsterName.json` と `MonsterNameBoss.json` を 1 設定項目に統合。`MonsterType` でモンスター種別を判定
- テストサーバーで発生した DungeonSyncData 解析失敗を修正
- ダメージ ID がルーティング表にない場合の個別ソート問題を修正
- 処理キュー監視変数の競合オーバーフローを修正

**Buff 関連**

- グループモード、独立モード「すべての Buff」の列数/行数などが設定できない問題を修正
- モンスターモニター 設定ページを追加。Boss Buff、Boss ヘイト関連データを表示
  - Boss Buff は自身が付与した Buff と任意のプレイヤーが付与した Buff を設定可能。例：自身付与はダメージ蓄積、ロロラなど自身の debuff。任意付与はブラジル、蜂巢、重傷などの状態
  - ヘイトリストは有効時にパーセンテージ形式で表示
  - 対応オーバーレイウィンドウを編集してレイアウト
- Buff 検索ですべての一致 Buff を表示。タイル表示で Buff 名 + ID + カスタムエイリアスを表示
- アイコンなし Buff、カスタム Buff などにプログレスバー透明度設定を追加。フォントシャドウを強化

**特別感謝**

- s3 データと一部ローカライズを提供してくれた YozoraHoshi に感謝

**注意**

- 0.0.2/0.0.3 を使用していた場合、0.0.7 を初めて使うときは `%LOCALAPPDATA%\resonance-logs-cn` 内の `resonance-logs-cn.db` を削除してからアプリを再起動してください
- npcap 処理ロジックを最適化済みのため、キャプチャには npcap を優先してください。WinDivert ドライバ署名期限切れによる AV 誤検知・隔離でアプリが正常動作しないのを避けられます

**特に注意**

コードを自分で改変して PR を出さず他人に配布する場合、アプリ名/バージョンなど原リポジトリ関連情報をすべて変更してください。原リポジトリ情報、特にアプリ名+バージョンを残さないでください
