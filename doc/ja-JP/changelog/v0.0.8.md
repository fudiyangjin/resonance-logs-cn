# Changelog v0.0.8

## 更新内容

### DPS 関連

- YozoraHoshi のデータに加えて s3 データを補完
- 双斧職業のアイコンと専精名の表示を追加
- ボス多目標スキルの占有率が誤っていた問題を修正
- 履歴で検索列を切り替えると検索欄の内容がリセットされる問題を修正
- 履歴の検索条件を永続化
- 履歴のシーン検索を修正し、名称を「シーン」に統一
- 履歴概要ページに「お気に入り」表示を追加
- 履歴再構築時にお気に入り済みレコードを除外
- 機械加工所ボスフェーズの異常リセットを修正
- 打桩モードを新規追加
  - リアルタイム画面で打桩モード用の木桩を選択可能（DPSメーター-リアルタイム-打桩モード）
  - 開始をクリックすると打桩待機状態に入る
  - 自身が初めてダメージを与えると打桩が正式に開始
  - その木桩が受けたダメージのみ記録（味方が同じ木桩に与えたダメージが記録されていても問題なし）
  - 時間経過で自動リセットし、履歴で具体的な占有率を確認可能
  - 手動リセット後は再び打桩待機状態となり、次のトリガーを待つ
  - ギルド退出、マップ切替などのシーンでは打桩モードを自動解除

### ライブモニター関連

- より安定した初期化用リスンリスト方式を採用
- カスタム監視でグループ追加と個別表示に対応
- 雷影剣士のスキルとリソースを追加
- スキル持続時間の設定と表示に対応。デフォルトで鎌車を追加
- ライブモニター画面の描画ロジックを最適化。毎フレーム全体のオーバーレイを再計算しないように変更

### モンスターモニター関連

- 仇恨リストの表示人数を制御可能に
- 非ボスの buff 監視を拡張する `ExtraBuffMonitoredMonsters.json` 設定を追加。デフォルトでエリート木桩 2 体
- 打桩モードを新規追加

### モジュール関連

- 列挙などのアルゴリズムをリファクタリング
  - 4 モジュールシーンで、800 モジュール列挙の CPU 時間を 87 秒から 43 秒に短縮
  - CUDA で 1000 モジュール列挙時間を 23 秒から 3 秒に短縮
- 5 モジュールフィルタに対応
- モジュール結果表示を最適化
- CUDA context が解放されず VRAM が解放されない問題を修正
- モジュール合計フィルタを追加。属性が 1 つのみのモジュールを直接除外

## 注意事項

### 互換性に関する注意

- 0.0.2 または 0.0.3 を使用していた場合、0.0.8 を初めて使う際は `%LOCALAPPDATA%\resonance-logs-cn` 配下の `resonance-logs-cn.db` を削除してからアプリを再起動してください
- npcap の処理ロジックを最適化済みのため、パケットキャプチャには npcap を優先してください。WinDivert ドライバの署名期限切れにより、ウイルス対策ソフトの誤検知・誤隔離が発生し、アプリが正常に動作しなくなる可能性があります
- モジュールフィルタについて、NVIDIA 以外の GPU では列挙戦略による解の品質補完は現時点では利用できません
- 0.0.8 を初めて使う際は、ライブモニター の「ライブモニターを有効化」と モンスターモニター の「モンスターモニターを有効化」をダブルクリック（一度オフにしてから再度オン）し、集約後のリスンリストを永続化してください。起動時のリスンリスト送信が不安定になる問題の対策です
