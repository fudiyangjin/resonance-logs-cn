# Changelog v0.0.9

## 更新内容

### DPS 関連

- リアルタイム、履歴にデータ小数桁表示設定を追加（DPSメーター-設定-リアルタイム/履歴-共通設定）
- w/億 への切り替え設定を追加（DPSメーター-設定-リアルタイム/履歴-共通設定）
- ミニマルテーマでダミー討伐モードボタンが自動非表示にならない問題を修正
- ダミー討伐モードボタン表示設定を ヘッダー設定 へ移動（DPSメーター-テーマ-リアルタイム-ヘッダー設定）。ダミーターゲット選択は従来位置
- モンスター死亡時に対応 Boss HP を送信しないよう変更
- リアルタイム DPS の回復スキルデータをルートで集約
- リセットロジックを変更。マップ切り替え後もリアルタイム DPS は即クリアせず、次のダメージパケット後に自動クリーンアップ（画面）。履歴は引き続きマップ切り替え時に保存。現在のリセットロジックを再掲：
  - マスターダンジョンは道中と Boss を自動分割、履歴 2 件
  - レイドは Boss ごとに独立分割
  - ワイプで自動リセット
  - 同一シーン内のリセットはすべて次の攻撃ターゲット後に自動（マスターダンジョン：道中クリア後 Boss 攻撃でリセット | レイド：新 Boss 攻撃ごと | ワイプ後：次の Boss 攻撃でリセット）
  - 今回追加：マップ切り替え後リアルタイム DPS は即クリアせず、次のダメージパケット後に自動クリーンアップ（画面）
- ダミー討伐モード UI 応答を最適化
- ライブ画面スキル詳細クリック応答を最適化
- 有効回復と有効 HPS の表示を追加

### ライブモニター 関連

- モンスター死亡時にモンスターのヘイト、Buff などの情報を送信しないよう変更
- 無限雷充電スタック数を 2 に調整。居合天賦必殺 CD を追加

### モジュール関連

- 5 モジュール非 NVIDIA GPU の GPU 演算をサポート

## 注意事項

### 互換性のお知らせ

- 0.0.2 または 0.0.3 を使用していた場合、0.0.4–0.0.9 を初めて使うときは `%LOCALAPPDATA%\resonance-logs-cn` 内の `resonance-logs-cn.db` を削除してからアプリを再起動してください
- npcap 処理ロジックを最適化済みのため、キャプチャには npcap を優先してください。WinDivert ドライバ署名期限切れによる AV 誤検知・隔離でアプリが正常動作しないのを避けられます
- 0.0.8 を初めて使うときは、ライブモニター の「ライブモニターを有効化」と モンスターモニター の「モンスターモニターを有効化」をダブルクリック（オフ→オン）してください。集約後の監視リストを永続化し、起動時の監視リスト送信不安定を解消
- 0.0.9 で有効回復フィールドを追加したため、旧履歴のエラーは正常。新しい履歴は正常
