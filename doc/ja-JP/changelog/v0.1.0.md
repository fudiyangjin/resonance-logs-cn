# Changelog v0.1.0

## 更新内容

### DPS 関連

- s3 更新設定を追加。一部シーン名を変更 @YozoraHoshi
- サーバー異常パケット送信によるデータ非表示問題の修正を試行
- リアルタイム DPS 画面のシーズン強度表示を同期

### ライブモニター 関連

- カスタム監視に因子エネルギー監視を追加（ライブモニター-カスタム監視-カスタムカウンタールール-ルールを新規作成-複数 source と複数 slot を選択-名称保存-カウンター追加で対応 slot 効果を選択）
  - 居合灵感因子 X1, X2, X4, X5。真实因子 X1, X2 を追加
  - 双斧灵感因子 X1, X2, X4, X9, X8。真实因子 X1, X3, X4, X5 を追加
  - ギター灵感因子 X10, X9, X5, X11。真实因子 X5 を追加
  - 鷹弓灵感因子 X7, X9, X11。真实因子 X5, X6 を追加
  - 氷法灵感因子 X1, X3, X4, X6, X7, X10。真实因子 X1, X3, X4, X5 を追加
  - 詳細設定は使用説明の ライブモニター を参照
- プリセット監視を追加：炎角-被動
  - slot1 はダメージ型専精カウント。絶対幸運の残りトリガー回数をカウント
  - slot2 は絶対幸運 CD。平穏期 15s、アクティブ発動時 5s
  - 「炎角-被動-幸運」（絶対幸運 Buff）の監視と slot1、slot2 を同一エリアに置くことを推奨。炎角被動の詳細表示。設定は使用説明の ライブモニター を参照
- スタック 1、持続時間 0 の Buff をユーザー指定監視後に正常表示
- キャラクターパネルに元素ダメージ関連属性監視を追加
- 更新後のキャラクターリソース異常を修正。リソース索引 ID に基づきプッシュ内容を表示

## 注意事項

### 互換性のお知らせ

- 0.0.2 または 0.0.3 を使用していた場合、0.0.4–0.1.0 を初めて使うときは `%LOCALAPPDATA%\resonance-logs-cn` 内の `resonance-logs-cn.db` を削除してからアプリを再起動してください
- npcap 処理ロジックを最適化済みのため、キャプチャには npcap を優先してください。WinDivert ドライバ署名期限切れによる AV 誤検知・隔離でアプリが正常動作しないのを避けられます
- 0.0.8 を初めて使うときは、ライブモニター の「ライブモニターを有効化」と モンスターモニター の「モンスターモニターを有効化」をダブルクリック（オフ→オン）してください。集約後の監視リストを永続化し、起動時の監視リスト送信不安定を解消

- コードを自分で改変して PR を出さず他人に配布する場合、アプリ名、バージョンなど原リポジトリ関連情報をすべて変更してください。原リポジトリ情報、特にアプリ名とバージョンを残さないでください
- よくある質問は使用マニュアルの html を参照
