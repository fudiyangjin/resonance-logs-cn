# Changelog v0.2.0

## 更新内容

### 音声読み上げ（新規）

- オフライン音声読み上げを追加：モデルをインストール後、音声のクローン、フレーズライブラリの管理ができ、トリガー時に生成済み音声を再生
- 音声モデルは Qwen3-TTS 0.6B ベースで、Q8_0 / F16 量子化 GGUF のみ対応。デフォルトでは「モデルとボイス」ページから Q8_0 版をワンクリックでダウンロード（ミラーソース対応）、変換済みモデルの手動インポートも可能
- 音声生成の前提条件：ゼロショットクローンには単一話者・BGM なしの 5～10 秒のクリーンな参考音声が必要。または自前の音声データで専用モデルをファインチューニング（Qwen3-TTS 0.6B CustomVoice ベース、GGUF としてエクスポートして参照。品質の上限がより高い）
- Buff モニター、モンスター Buff、カスタムカウンター、Boss DBM、ダンジョンギミックでインラインに音声バインドを設定可能
- 読み上げ一覧ページで全バインドを集約し、不足している音声をワンクリックで生成
- CPU / Vulkan アクセラレーションと再生キュー戦略（低優先度を破棄 / 高優先度で割り込み）をサポート
- Buff / モンスター Buff の「付与時 / 解除時」カスタムテキストで `${阶数}`（または `${remodelLevel}`）プレースホルダーをサポート。ファンタジー 0～5 段階に応じて必要な音声を生成
- フレーズライブラリで未参照フレーズのワンクリック整理をサポート
- 使い方は[音声読み上げガイド](../features/voice/README.md)を参照

### ロードアウト

- ロードアウト（総合プロファイル）機能を強化：スキルモニター、モンスターモニター、DPS のサブプロファイルを組み合わせ、ワンクリックで一括切り替え
- DPS 関連設定を分割してロードアウト管理下に移動。ロードアウトと一緒にエクスポート・共有可能
- 光盾、リカバリー、ブロック、アースフォート、スマイト、コンチェルトの職業プリセットを追加。初回起動時にプリセットを選択してすぐに開始可能
- ロードアウトのインポート / エクスポートをサポート

### DPS / 履歴

- デスリプレイに被弾参加者の Buff スナップショット表示を追加 @Sanheiii
- Live 画面にキャラクターが直近に使用したファンタジーの表示を追加（DPSメーター → 設定 → ライブ → 幻想アイコンを表示 で有効化）
- 背景要素設定とツールチップのスタイルを統一

### ライブモニター関連

- カスタム Buff アイコンをサポート（任意の Buff に画像を設定可能）
- Buff モニターで音声読み上げを設定可能（付与時 / 解除時 / N 秒前など）
- ファンタジーが付与した Buff の段階を解析できる場合、オーバーレイの名称を「名称 | n段階」として表示

### モンスターモニター関連

- Boss DBM、モンスター Buff で音声読み上げを設定可能
- Buff 名ドキュメントを調整

### ダンジョンギミック関連

- ギミックの音声読み上げバインドをサポート（対応マップごとに設定可能）
- 死亡マーカーのスタイルをカスタマイズ可能（色、透明度、× マーク）
- ギミック音声読み上げ設定エリアはデフォルトで折りたたみ

### データとローカライズ

- 正式サーバー 1.2.43804.0 データ更新 @Yozora_Hoshi
- 日本語 / 英語データを補充し、ファンタジースキルを補完

## 注意事項

### 互換性に関する注意

- 0.0.2 または 0.0.3 を使用していた場合、0.0.4〜0.2.0 を初めて使う際は `%LOCALAPPDATA%\resonance-logs-cn` 配下の `resonance-logs-cn.db` を削除してからアプリを再起動してください
- 0.1.5 ではインスタンス保存 key（uid -> uuid）とパケットキャプチャ処理を変更しているため、異常があればログを添えて報告してください
- 0.1.5 で追加した因子の一部は十分に自測できていないため、異常があれば報告してください
- 音声読み上げには別途モデルのダウンロードが必要です。初回のフレーズ生成には時間がかかる場合がありますが、生成後の再生ではモデルを読み込みません
