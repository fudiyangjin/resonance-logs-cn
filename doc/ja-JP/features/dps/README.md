# DPS 測定

リアルタイムダメージ統計、履歴、カスタム列設定の説明です。

## 概要

DPS 測定はネットワークキャプチャで戦闘データを解析し、次をサポートします。

- **リアルタイム統計**：戦闘中にダメージ、秒間ダメージなどを即時更新
- **履歴**：過去の戦闘を保存
- **多次元データ**：ダメージ、回復、被ダメ（DPS / HPS / TPS）

| [設定](./settings.md) | DPSメーター → 設定 | 列表示、更新頻度、ネットワーク、ショートカット |

## ダミー討伐モードの使い方

ダミー討伐モードは固定ダミーシーンで 1 ラウンドの出力を単独記録するのに適しています。終了後 **ライブ画面は現在のデータで凍結**し、すぐにスキル構成比と秒間ダメージを確認でき、[履歴](./history.md) を開かなくても済みます。

### 入口

- **訓練用ダミーボタンを表示**：`DPSメーター → テーマ → ライブ → ヘッダー設定 → 訓練用ダミーボタンを表示`（[テーマ](./themes.md) 参照）
- **開始/終了**：ライブウィンドウヘッダーの照準ボタン

設定でダミータイプを事前選択する必要はありません。ダミー討伐モードをオンにしたあと、**対応ダミーに初めてダメージを与えたとき**に **精英敵方ダミー** か **精英守護ダミー** かを自動識別・ロックします（実際に命中したターゲットで決まります）。

### 状態遷移

```
待機（ダミー討伐待機） → 討伐中 → 討伐終了
         ↑                              |
         └──────── 「リセット」クリック ──────────┘
```

- **ダミー討伐待機**：モードオン、ダミーへの初ダメージ待ちでターゲット自動ロック
- **討伐中**：現在のダミーをロックし、そのターゲットの戦闘データのみ集計
- **討伐終了**：今ラウンドの統計が確定し、ライブは更新停止。次ラウンドの開始は任意

次ラウンドは先に **リセット** で **ダミー討伐待機** に戻し、ダミーにダメージを与えると再び **討伐中** になります。討伐終了後の再攻撃では自動で新セグメントは始まりません。

### 使用フロー

1. ヘッダー設定 で 訓練用ダミーボタンを表示 を有効にし、ライブウィンドウで照準をクリックしてダミー討伐モードをオン（**ダミー討伐待機**）。
2. **精英敵方ダミー** または **精英守護ダミー** に初ダメージ後、自動ロックして **討伐中**。
3. 討伐中はロックしたダミー関連データのみ累計。パーティが同ダミーに与えたダメージが統計に含まれることは正常です。
4. 1 ラウンドの時間上限で **討伐終了**、ライブ凍結で確認。今ラウンドのデータは [履歴](./history.md) にも書き込まれます。
5. 確認後 **リセット** で **ダミー討伐待機** に戻し、準備ができたらダミー攻撃で次ラウンド開始。

### 注意事項

- **本人**が対応ダミーに与えた初ダメージのみがロックと正式記録の開始をトリガーします。
- **討伐終了** 後の再攻撃では新セグメントは始まらず、**リセット** 後に攻撃が必要です。
- **討伐中** でも **リセット** で早期終了し **ダミー討伐待機** に戻せます。
- ダミー討伐モードをオフ（照準を再クリック）すると討伐状態を終了します。
- ギルド脱退、マップ切り替え、チャンネル変更などでシーンが変わると自動でモード解除。再度オンにしてください。

## データ指標の説明

### スキル欄の構成

DPS スキル欄はクライアントの DPS ロジックに従います。各スキル下に複数のダメージソースがあり、各ヒットは固定ルールで**ダメージ ID** に連結され、ダメージ ID で 1 スキルに集約表示されます。一部スキルを展開すると複数の明細行が見えます。

![スキル欄 - 集約前](../../../shared/img/dps/dps_1.png)

![スキル欄 - 複数行を展開](../../../shared/img/dps/dps_2.png)

集約ルールの概要：

1. **生ダメージ行**：各ヒット 1 行。スキル ID、ダメージソース、ヒットイベント ID など
2. **集約表示**：同一ダメージ ID の行を 1「スキル」に。展開で子行を表示

### 真秒間ダメージと活動戦闘時間

- **秒間ダメージ**：戦闘全体の実時間で計算
- **真秒間ダメージ**：「実際にダメージが発生した活動時間」で計算

**DPSメーター → テーマ → ライブ → ヘッダー設定** で「アクティブ戦闘時間を表示」にチェックすると活動時間を確認できます。詳細は [テーマ](./themes.md)。

### 回復 / 被ダメ列

回復・被ダメモードも同様の列構造で、HPS（秒間回復）、TPS（秒間被ダメ）およびクリティカル、ラッキーなどを集計します。

## リセットロジック

戦闘統計の区切りとリセットは次のとおりです。

| シーン | 動作 |
|------|------|
| **マップ切り替え** | シーン/マップ変更時：今回に戦闘データがあれば履歴 1 件として保存。ライブは即クリアせず、**次にダメージを与えた後** 自動リセットして新シーン統計開始 |
| **マスターダンジョン** | 道中と Boss を自動分割。履歴 2 件：道中 1、Boss 1 |
| **レイド** | 各 Boss 独立分割。Boss ごとに履歴 1 件 |
| **ワイプ** | ワイプ時に現在統計を自動リセット |

### マップ切り替えの補足

- **データがある場合のみ処理**：切り替え前に有効な戦闘統計がなければ（未戦闘など）、余分な履歴書き込みや上記の遅延リセットは発生しません。
- **ライブ vs 履歴**：履歴はマップ切り替え時に保存。ライブ DPS は再攻撃まで前回データを残し、切り替え直後に数値が消えて誤解されないようにします。

同一シーン内のリセットはいずれも「次の攻撃ターゲット後」に自動発動：

- **マスターダンジョン**：道中クリア後、Boss への初攻撃で自動リセット。Boss 区間を単独集計
- **レイド**：新 Boss への攻撃ごとに自動リセット
- **ワイプ後**：次の Boss 攻撃で自動リセット
