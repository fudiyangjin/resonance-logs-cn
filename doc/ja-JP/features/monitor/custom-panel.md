# カスタム監視

**ライブモニター → カスタムモニター** タブに対応します。

複数の**テキスト監視エリア**を作成し、オーバーレイで Buff やカウンターをプログレスバーなどで表示できます。同一 Buff またはカウンタースロットはすべての監視エリアで**グローバルに一意**（複数エリアへ重複追加不可）。

![カスタムパネルの例](../../../shared/img/monitor/buff_3.png)

## 通常カスタムエリアと因子エリア

| 種類 | 作成方法 | 用途 |
|------|----------|------|
| **通常カスタムエリア** | **監視エリアを新規作成** | Buff、プリセット/カスタムカウンターを手動追加して自由に組み合わせ |
| **因子設定エリア** | **因子エリアを新規作成** | 現在キャラの**シーズン育成因子**関連進捗を自動表示。Sources/Slots の手動設定不要 |

### 因子エリア（因子エリアを新規作成）

**因子エリアを新規作成** をクリックすると、現在キャラに装備された因子パスに基づきオーバーレイにエントリを自動生成。典型的には（実際の装備によります）：

- **真实因子 A / B エネルギー**（カウント進捗）
- **真实因子 A / B Buff 効果**（関連 Buff 持続時間など）

**同期と変更：**

- **マップ切り替え**後に全体データを 1 回同期。以降ゲーム内で**因子パス切り替え**や**装填因子変更**時、因子エリア表示も更新され、エリアの再作成は不要
- 因子エリアは通常エリアのように Buff / カウンターを手動「追加」できません。エントリはシステムが自動維持

**表示名称：**

- **因子エネルギーなどカウント系**エントリ：因子エリア選択後、**因子表示名** で slot（名称 / 説明）を検索してカスタム名称を設定。slot テンプレートで保存され、プリセット切り替え後も継続
- **因子 Buff 効果**エントリ：[Buff 監視 → Buff 別名設定](./buff.md#buff-エイリアス) で名称変更（エイリアスはグローバル有効）

因子エリアも通常エリア同様 **監視エリア名称**、**スタイル**（行間、フォント、色など）を設定でき、[ウィンドウを有効化](./overlay.md) の **カスタム監視エリア** スイッチで全体表示/非表示。

## 通常カスタムエリア

### 監視エリアとスタイル

- 各通常エリアはオーバーレイで**独立テキスト領域**。個別にドラッグ、リサイズ可能（[ウィンドウを有効化](./overlay.md) · オーバーレイレイアウトを編集）
- **監視エリア名称**、**現在の監視エリアスタイル**（行間、フォントサイズ、名称と数値の間隔、名称/数値/プログレスバーの色と透明度）をエリアごとに保存

### カウンター

「Buff、ダメージタイプに関連」するステートマシンをサポート。Buff で維持されない特殊トリガーをカウントする用途。例：

- 幻想衝撃のカウント
- 超然トリガーのカウント

通常カスタムエリア選択後、**カウンターを追加** で設定。カウンタースロットはグローバル一意。プリセットルールとカスタムルールを一覧表示。

### Buff を追加

**現在の**通常カスタムエリアのテキスト領域にのみ追加。Buff 検索 grid から選択（他エリアで使用中の Buff は重複追加不可）。

### 現在のグループエントリ

追加済み Buff / カウンターの表示名（カウンター）、並び順（上へ/下へ）、削除を調整可能。

## カスタムカウンタールール

組み込みプリセットに加え、**カスタムカウンタールール**を自作可能。どのゲームイベントを **Sources（ソース）** としてカウントを進めるか、どの **Slots（スロット / 効果位置）** でパネルに進捗や状態を表示するかを指定。入口：**ライブモニター → カスタムモニター → カスタムカウンタールール**。

**一般的な流れ：**

1. **ルールを新規作成**。**Sources** で必要なソースを選択または追加（複数可）
2. **Slots** でカスタムパネルに紐づけるスロットを選択（複数可）。ルールロジックと 1 対 1 対応
3. **ルール名称** を入力して**保存**
4. 通常カスタムエリアで **カウンターを追加**、保存したルールの対応 Slot を選択。同時に見たい **Buff 監視** と **カウンター** は**同一オーバーレイ領域**に置くと一目で対照可能

## 応用例

### 赤紅流（双斧因子）

**赤紅流** を例に、双斧関連因子ソースでカウントを駆動し、指定スロットで因子エネルギーとトリガー関連進捗を表示。

**1. Sources と Slots を設定してルール保存**

- **ルールを新規作成** をクリック
- **Sources** で **双斧X9**、**双斧X8** を選択（カウントソース）

![赤紅流ルール：Sources で双斧X9、双斧X8 を選択](../../../shared/img/monitor/buff_7.png)

- **Slots** で **双斧X4**、**双斧X5** を選択。ルール名称入力後 **保存**

![赤紅流ルール：Slots で双斧X4、双斧X5 を選択して保存](../../../shared/img/monitor/buff_8.png)

**2. レイアウト：因子トリガー Buff と因子エネルギーカウントを同一エリア**

オーバーレイまたはカスタムパネルレイアウトで、**因子トリガー関連 Buff** と **因子エネルギーカウント** を**同一表示領域**に配置。「トリガー有無」と「エネルギー/進捗」を同時に確認しやすくなります。

### プリセット監視：炎角-被動

組み込み **炎角-被動** プリセットルール。カスタムパネルで炎角被動と「絶対幸運」関連状態を観察。カウンターとスロットをゼロから設定する必要なし。

**ルール要点：**

- **Slot 1（ダメージ型専精カウント）**：スキル詠唱完了ごとに回数を累計。「閾値まであと何回」形式で表示。**絶対幸運** の残りトリガー回数を把握しやすい
- **Slot 2（絶対幸運 CD）**：被動 Buff と連動して CD 進捗を表示。**平穏期** 約 **15 s**、特定条件（例：アクティブ関連 Buff 発動時）では約 **5 s** のタイマー表示

**推奨用法：**

1. **カスタムカウンタールール** で **炎角-被動** プリセットを選択または保存後、カスタムパネルで**カウンターを追加**、対応 **Slot 1、Slot 2** を選択
2. 同時に [Buff 監視](./buff.md) で **「炎角-被動-幸運」**（絶対幸運 Buff）を追加。上記 2 カウンターと**同一表示領域**に置くと、オーバーレイで幸運 Buff とカウント、CD の連携情報を同時確認可能
