# Changelog v0.0.7

## Logs

**dps 相关**

- 修复 npcap 某些场景下无法正常抓包的问题, 扩展以太帧的处理范围
- 新增 s3 数据, 部分数据本地化 @YozoraHoshi
- 移除发送没有意义的 boss 死亡消息
- 全量同步数据前清理部分数据, 保持列表干净
- 将 MonsterName.json 和 MonsterNameBoss.json 合并为一个配置项, 通过 MonsterType 判断怪物类型
- 修复测试服出现的 DungeonSyncData 解析失败的问题
- 修复伤害 id 不在路由表时单独排序的问题
- 修复处理队列监控变量出现的竞争溢出问题

**buff 相关**

- 修复分组模式, 独立模式全部 buff 的列数行数等不可设置的问题
- 新增怪物监控设置页, 用于展示 boss buff, boss 仇恨相关数据
    - 对于 boss buff 可以设置由自身触发的 buff 和任意人触发的 buff. 例如自身触发的 buff 应该设置为伤害叠加, 罗罗拉等自身对敌方的 debuff. 任意人触发的 buff 应该设置为巴西, 蜂巢, 重伤等状态
    - 仇恨列表如果启动以百分比形式展示
    - 通过编辑对应遮罩窗口进行具体布局
- buff 搜索展示所有匹配到的 buff, 并展示样式为平铺模式, buff 名 + id + 自定义别名
- 无图标 buff, 自定义 buff 等增加支持进度条透明度设置, 增强字体阴影

**特别感谢**

- 感谢 YozoraHoshi 提供的 s3 数据和部分本地化

**注意**

- 如果之前使用过 0.0.2/0.0.3 第一次使用 0.0.7 需要清理 `%LOCALAPPDATA%\resonance-logs-cn` 目录下的 `resonance-logs-cn.db` 再重新启动应用
- 由于已经优化了 npcap 的处理逻辑, 请优先使用 npcap 进行抓包, 避免因 WinDivert 驱动签名过期，引发杀毒软件误报、误隔离，导致应用无法正常运行
