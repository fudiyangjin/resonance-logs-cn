# Changelog v0.1.5

## 更新内容

### DPS 相关

- 优化程序整体时延响应
- 移除 WinDivert 逻辑，重构抓包逻辑
- 修复幸运率的计算和展示
  - 现在会单独统计每个技能的触发幸运率
  - 幸运率计算会排除“幸运一击”的命中数
- Tank 新增格挡率、幸运格挡率展示
- Tank 实时与历史记录会先按怪物聚合展示，点击后查看对应怪物的技能承伤详情
  - 总计
  - 怪物 A 承伤详情
  - 怪物 B 承伤详情
- 修改打桩模式逻辑
  - 移除前端目标设置，开启打桩模式后自动识别精英攻击木桩或守护木桩
  - 打桩流程改为由用户决定下一次打桩时机：`待打桩 -> 打桩中 -> 打桩结束`，点击重置后回到待打桩
  - 打桩结束后 live 界面不再刷新，方便直接点击查看，不必专门打开历史记录
- 新增直到妖尾版本的正式服数据，包括 buff、场景、新增幻想等，感谢 @YozoraHoshi

### 实时监控相关

- 自定义监控新增“新建因子区”，可自动配置并展示当前角色的因子效果
  - 真实因子 A 能量情况
  - 真实因子 B 能量情况
  - 真实因子 A Buff 效果
  - 真实因子 B Buff 效果
- 因子 Buff 效果重命名位置：实时监控 -> Buff 监控 -> Buff 别名设置
- 真实因子计数重命名位置：对应因子区的因子显示名搜索与设置
- 修复丹佛自伤没有进入 Tank 记录，导致相关计数错误的问题
- 新增真实因子

  - 巨刃 X1, X2, X4, X5, X6
- 新增虚妄因子

- 增加聚合乐章、万众瞩目、升格、无限狂想的 CD

### 怪物监控相关

- 废弃 `ExtraBuffMonitoredMonsters.json` 配置，怪物 buff 与仇恨现在会自动显示为你攻击的主目标
- 怪物监控新增队友 buff 监控
- 优化实时监控幽灵层展示
- 怪物 buff 新增“仅自身施加”展示，方便快速找到怪物需要监听的 buff

## 注意事项

### 兼容性提醒

- 如果之前使用过 0.0.2 或 0.0.3，第一次使用 0.0.4-0.1.4 需要清理 `%LOCALAPPDATA%\resonance-logs-cn` 目录下的 `resonance-logs-cn.db`，再重新启动应用
- 0.1.5 移除了 WinDivert 抓包逻辑，之前使用 WinDivert 的用户请下载 Npcap
- 0.1.5 修改了实例存储 key（uid -> uuid）与抓包逻辑，整体改动较大，若有异常请及时联系并提供日志
- 0.1.5 新增了统计字段，旧历史记录不再兼容解析展示
- 0.1.5 新增的部分因子未完整自测，如有异常请及时反馈

### 特别注意

- 如果你自己修改了代码又不想提交 PR，在分享给其他人时请改掉应用名、版本号等原版相关信息，尤其是应用名和版本号
- 关闭按钮现在会隐藏到右下角，可以将隐藏栏内容拖到右下角显示，避免窗口程序过多时无法快速找到
- 常见问题见使用手册中的 HTML
