# 语音播报

对应工具箱中的 **语音播报**。离线克隆或微调音色、维护固定短句，并在 Buff / 机制触发时播放**已生成**的音频（播放阶段不再加载模型）。

使用上分为两大块：

1. **通用配置流程**：在 Buff / 机制等业务页勾选事件、配置文案，决定「什么时候播、播什么」。
2. **生成播报内容**：选择音色来源（0 样本克隆 / 微调），把文案合成为本地音频。

## 通用配置流程

### 前提

1. 打开 **语音播报 → 模型与音色**，下载安装语音模型（也可手动导入已转换模型）。
2. 打开 **总览与播放**，开启 **启用语音播报**，按需调整音量、队列策略与生成加速（自动 / CPU / Vulkan）。

### 在 Buff 界面配置播报

以最常用的 **实时监控 → Buff 监控** 为例：展开已监控 Buff 下方的 **语音播报** 区块，搜索已监控 Buff 并添加播报规则。

![Buff 界面配置播报示意](../../../shared/img/voice/voice_1.png)

1. 点击 **搜索已监控 Buff 并添加语音播报**，把要播报的 Buff 加入列表。
2. 勾选触发事件：**生效时播报** / **剩 x 秒播报** / **失效时播报**。
3. 为每个事件选择文案来源：
   - **自动生成**：按 Buff 名称与事件类型生成默认文案（可预览）
   - **自定义文本**：手写播报内容，例如「蒂娜 CD 结束了」；幻想相关 Buff 还可插入阶数占位符（见下）
   - **短语库引用**：复用短句库中已有短句
4. 点 **试听** 验证；若提示待生成，说明音频还没合成，见下文「生成播报内容」。

> 右上角会显示该区块已配置的播报数量，可随时 **移除播报**。

### 自定义文本中的幻想阶数

部分幻想施加的 Buff，其来源幻想有 **0～5 阶**。若希望播报里带上阶数，可在 **生效时 / 失效时** 的自定义文本中写入占位符（「剩 x 秒」不支持）：

| 占位符 | 说明 |
|--------|------|
| `${阶数}` | 中文习惯写法 |
| `${remodelLevel}` | 与上等价 |

示例：自定义「蒂娜${阶数}已生效」→ 实际 5 阶时播报「蒂娜5阶已生效」。

说明：

- 仅 **Buff 监控** 与 **怪物 Buff 监控** 的「生效时 / 失效时」支持；计数器、Boss DBM、副本机制、以及「剩 x 秒」事件不会展开阶数。
- 配置后会按需生成 **0～5 阶各一条**，外加一条「未知阶数」时的兜底文案（占位符去掉），并非所有 Buff 预先批量生成。
- **试听** 会按 5 阶示例展开占位符；实战播报按当次触发时解析到的幻想阶数选对应音频，解析不到则用兜底文案。
- 改完自定义文本后，请到 **播报总览** 确认状态并 **一键生成缺失语音**。

悬浮窗 Buff 名称也会在能解析到来源幻想阶数时显示为「名称 \| n阶」（与是否配置语音无关）。

### 其他配置途径

语音规则都内联在各功能页配置，不在「播报总览」里新建：

| 配置位置 | 典型事件 |
|----------|----------|
| 实时监控 → Buff 监控 | 生效时 / 失效时 / 剩 x 秒 |
| 实时监控 → 自定义监控（计数器） | 达到目标时 |
| 怪物监控 → Buff 监控 | 生效时 / 失效时 / 剩 x 秒 |
| 怪物监控 → Boss DBM | 触发时 |
| 副本机制 → 机制语音播报 | 各地图已接入的机制预警 |

### 播报总览

**语音播报 → 播报总览** 只读汇总以上所有配置（按 Buff 监控、怪物 Buff、计数器、Boss 机制、副本机制分组）：

- 状态为 **待生成** 时，先在模型页装好模型并选好音色，再点 **一键生成缺失语音**。
- 可通过「前往 xxx」跳转到对应配置页修改绑定。

## 生成播报内容

播报文案配置好后，需要把文本合成为音频。音色来源有两种方式：**0 样本克隆**（简单快捷）与 **微调**（效果上限更高）。

### 方式一：克隆音色（0 样本）

在 **短句库 → 批量生成** 中选择 **克隆音色**，无需训练，一段参考音频即可：

![克隆音色批量生成示意](../../../shared/img/voice/voice_2.png)

1. 选择 **使用已有音色**（复用已保存的音色档案），或 **新建音色**：
   - 填写新音色名称，选择参考音频（WAV）；
   - 可选勾选 **保留参考音频文件**。
2. 勾选待生成短句（支持全选），点 **开始生成**，生成后立即生效。

**参考音频建议**：选择 **5~10 秒的角色干音**——单一说话人、清晰、无背景音乐、无混响。参考音频越干净，克隆出的音色越像。

短句状态含义：

- **已生成**：可直接播放
- **需重新生成**：音色或文本变更导致旧音频过期
- **未生成**：尚无有效音频

> 首次生成可能较慢；生成完成后触发播报只播放本地音频，不再加载模型。

短句库还提供 **清理未引用短语**：移除播报或修改文案后残留的自动生成短语（及其音频）可一键删除。清理会检查全部监控子方案（不只当前激活的），仅删除没有任何方案引用的自动短语，手动创建的短句不受影响。

### 方式二：微调音色（进阶）

0 样本克隆只靠几秒参考音频，相似度与稳定性有限；如果追求更高还原度，可以用目标角色的语音数据**微调**一个专属模型，导出后在 **模型与音色** 页引用为 **微调音色**。

大致流程：

1. **获取素材**：B 站视频可用 [bilibilidownloadtool](https://github.com/NANblogink/bilibilidownloadtool) 下载。
2. **提取音频**：用 FFmpeg 转成无损 PCM WAV（保留原始采样率，不要先压成 MP3）。
3. **分离人声**：用 [Ultimate Vocal Remover](https://github.com/Anjok07/ultimatevocalremovergui) 剥离背景音，导出 `Vocals` 干声，并完整试听检查。
4. **制作数据集**：按静音切分、重采样到 24kHz，用 ASR 转写并人工校对文本。
5. **训练**：生成音频离散码与 speaker embedding，微调 Qwen3-TTS 0.6B CustomVoice（推荐在 Docker 中运行，预构建镜像 `ghcr.io/mozi1924/qwen3-tts-easyfinetuning`）。
6. **导出部署**：导出 GGUF 部署目录，在应用内引用为微调音色。

数据与环境要求：

- 只使用自己拥有或已获授权的声音数据，且一个数据集只包含一个目标说话人。
- 建议准备 **10~30 分钟**干净有效语音；少量数据也能训练，但相似度与稳定性更弱。
- 训练建议 NVIDIA GPU（显存 ≥ 16GB），Windows 下推荐 WSL2 + Docker Desktop。

## 播放与队列

- **启用语音播报** 关闭后，所有触发规则都不会播放。
- 每个播报事件都可以单独设置优先级档位；未设置时使用最低优先级。同优先级的播报不会互相打断，而是按队列逐条播放。
- 队列策略：
  - **队列已满时丢弃低优先级**：高峰时避免堆积
  - **高优先级可打断当前播放**：只有优先级严格高于当前播报的事件才会抢占
- 可随时 **停止播放**。

## 与监控方案的关系

总方案（Loadout）会一并关联实时监控、怪物监控等设置；语音绑定随对应监控子方案保存。切换总方案后，请到 **播报总览** 确认音频是否仍为「已就绪」（换机或导入方案后可能需要重新生成）。

## 常见问题

| 现象 | 处理建议 |
|------|----------|
| 试听提示播放失败 | 确认模型已安装，且短句 / 绑定已生成音频 |
| 后端显示「不可用」 | 安装 Vulkan 运行时后点总览 **刷新**；或改用 CPU / 自动 |
| 微调音色不可用 | 检查部署目录是否移动；使用「重新定位」与「完整性检查」 |
| 机制页看不到可播报项 | 当前地图尚未接入机制语音；见副本机制页说明 |
| 生成被占用 | 等待当前下载 / 生成完成，或取消后再试 |
| 自定义了 `${阶数}` 但播报没有阶数 | 确认事件是「生效时 / 失效时」且文案为自定义；到播报总览生成各阶变体；部分幻想 Buff 的施加者不是召唤体时，阶数可能暂时解析不到，会播兜底文案 |

更多引擎与模型清单细节见仓库 `docs/VOICE_MODEL_MANIFEST.md`（面向开发 / 发布）。
